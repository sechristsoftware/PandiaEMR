Readme - 360 | SpeechAnywhere Services (Browser edition): How to enable microphone button events

360 | SpeechAnywhere Services (Browser edition) provides components which you can use to enable button events for certain microphones in Microsoft Internet Explorer.

Important: In the current release this functionality is provided in demo quality only; do not use it in a production environment.

Functionality

If button events are enabled, 360 | SpeechAnywhere Services reacts to microphone buttons in the following way:

  • Pressing Record on the microphone starts/stops recording.
  • Pressing Fast Forward/Fast Rewind on the microphone moves the speech focus to the next/previous text control.

Deployment and activation procedure

To enable microphone button events, do the following:

  1. Place the following binaries into a folder on your web server:
    • NuanceSpeechAnywhereBrowserHelper.dll
    • PIA.SmXAudioLib.dll
    • NuanceSpeechAnywhereBrowserHelperSFX.exe
  2. Add the following object tag to each page that embeds the Nuance SpeechAnywhere SDK and that you want to support microphone buttons for:

<object height="0" width="0" classid="<path>/NuanceSpeechAnywhereBrowserHelper.dll#Nuance.SpeechAnywhere.BrowserHelper" id="NUSABrowserHelper">

If NuanceSpeechAnywhereBrowserHelper.dll and the other binaries are in the same location as the web page, specifying the path is not required.

Note: The drivers for USB SpeechMike and PowerMic button support are embedded in the NuanceSpeechAnywhereBrowserHelperSFX.exe package. For all other supported microphones (Olympus and Grundig) you must get the USB driver package from the respective vendor and install it separately on each client.

Troubleshooting

If 360 | SpeechAnywhere Services does not react to button presses on the microphone, make sure that Microsoft Internet Explorer allows the execution of Active-X controls implemented in .NET:

  • Your .Net installation must allow execution of code from the web server. This can be enabled using the following command line:

    • "%WINDIR%\Microsoft.NET\Framework\v2.0.50727\CasPol.exe" -q -machine -addgroup 1. -site localhost FullTrust

    localhost must be replaced with the name of the web server.

  • Internet Explorer security settings must be set to "low" for the web server or security settings must be set to a custom level that allows execution of Active-X controls implemented in .NET.
